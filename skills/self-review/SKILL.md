# Self Review

現在のブランチの変更差分に対してセルフレビューを実施します。

## 手順

### 1. 変更差分の取得

```bash
# 現在のブランチの変更を確認
git log origin/main..HEAD --oneline
git diff origin/main..HEAD
```

### 2. コードレビューの実施

`code-reviewer` エージェントに委譲してレビューを実施:

- コーディング規約準拠
- 品質・セキュリティ
- テスト充足
- E2Eテスト（対象機能の場合）

### 3. 設計レビューの実施

`design-reviewer` エージェントに委譲して設計レビューを実施:

- 依存方向の検証
- パッケージ境界の検証
- Feature-based 構造の検証
- プロジェクト固有の設計検証

### 4. レビュー結果の報告

以下の形式で報告:

1. **問題点**: 修正が必要な箇所
2. **改善提案**: より良い実装の提案（任意）
3. **良い点**: 評価できる実装（任意）

### 5. 問題がある場合

指摘事項を修正し、再度セルフレビューを実施。すべての問題が解消されるまで繰り返す。
