# origin/main リベース指示書

**あなたは現在のブランチをorigin/mainにリベースし、コンフリクトを解消するエージェントです。**

---

## 実行手順

### Phase 1: 現在の状態確認

1. **現在のブランチとステータスを確認**
   ```bash
   git branch --show-current
   git status
   ```
   - 未コミットの変更がある場合は警告を出す

2. **origin/mainを最新に更新**
   ```bash
   git fetch origin main
   ```

### Phase 2: リベース実行

1. **リベースを開始**
   ```bash
   git rebase origin/main
   ```

2. **結果を確認**
   - 成功した場合はPhase 4へ
   - コンフリクトが発生した場合はPhase 3へ

### Phase 3: コンフリクト解消

コンフリクトが発生した場合、以下を繰り返し実行：

1. **コンフリクトファイルを確認**
   ```bash
   git diff --name-only --diff-filter=U
   ```

2. **各コンフリクトファイルの内容を確認**
   - ファイルを読み込んでコンフリクトマーカーを確認
   - `<<<<<<<`, `=======`, `>>>>>>>` を探す

3. **コンフリクトを解消**
   - 両方の変更の意図を理解する
   - 適切にマージして解消
   - ファイルを修正

4. **解消したファイルをステージング**
   ```bash
   git add <解消したファイル>
   ```

5. **リベースを続行**
   ```bash
   git rebase --continue
   ```

6. **まだコンフリクトがあれば手順1に戻る**

### Phase 4: 完了確認

1. **リベース完了を確認**
   ```bash
   git status
   git log --oneline -5
   ```

2. **origin/mainとの差分を確認**
   ```bash
   git log origin/main..HEAD --oneline
   ```

---

## コンフリクト解消の判断基準

- **機能的な変更**: 両方の意図を保持するようにマージ
- **同じ箇所への変更**: 最新の要件や設計に基づいて判断
- **import文**: 必要なものを全て含める
- **型定義**: 互換性を保つように統合

---

## リベース中断時

判断が難しい場合やユーザー確認が必要な場合：

```bash
git rebase --abort
```

でリベースを中断し、ユーザーに状況を報告してください。

---

## 作業完了時

以下を報告してください：
1. リベースの成否
2. 解消したコンフリクトの数とファイル
3. 各コンフリクトの解消方法の概要
4. 現在のブランチの状態（origin/mainからの差分コミット数）
