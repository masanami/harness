# テスト実行・分析

プロジェクトのテストを実行し、結果を分析します。

## 手順

### 1. テスト環境の確認

プロジェクトのテスト設定を確認:
- `package.json` のテストスクリプト
- テスト設定ファイル（`jest.config.*`, `vitest.config.*`, `pytest.ini` 等）
- AGENTS.md のテスト関連の記述

### 2. テスト実行

プロジェクトで定義されたテストコマンドを実行。

引数が指定された場合:
| 引数 | 動作 |
|------|------|
| ファイルパス | 指定ファイルのテストのみ実行 |
| `--watch` | ウォッチモードで実行 |
| `--coverage` | カバレッジレポート付きで実行 |
| 未指定 | 全テスト実行 |

### 3. 結果分析

テスト結果を分析:
- パスしたテスト数
- 失敗したテスト数
- スキップされたテスト数

### 4. 失敗テストの診断

失敗したテストがある場合:
1. エラーメッセージを確認
2. 失敗の原因を特定（コード変更、環境問題、テスト自体の問題）
3. 修正方法を提案

### 5. レポート

```
## テスト結果

### サマリー
- 総テスト数: X
- パス: X
- 失敗: X
- スキップ: X

### 失敗テスト（ある場合）
1. テスト名
   - エラー内容
   - 推奨対応

### 次のアクション
- （必要に応じて）
```
