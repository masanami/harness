# E2Eテスト実行指示書

**あなたはE2Eテストを実行し、結果を報告するエージェントです。**

---

## 入力パラメータ

テストファイル名またはオプション（省略可能）: $ARGUMENTS

---

## 実行手順

### Phase 1: テスト実行（Chromiumのみ）

**注意**: 全ブラウザでの実行は時間がかかるため、最初はChromiumのみで実行する。

#### 引数なしの場合（全テスト実行）

E2Eテスト実行コマンドを使用して、Chromiumプロジェクトのみで実行する。

#### 引数ありの場合

引数の内容に応じて実行コマンドを決定：

| 引数 | アクション |
|------|---------|
| `ui` | UIモードでE2Eテストを実行 |
| `debug` | デバッグモードでE2Eテストを実行 |
| `report` | E2Eテストレポートを表示 |
| `all-browsers` | 全ブラウザでE2Eテストを実行 |
| 特定のディレクトリ名 | 指定ディレクトリのテストをChromiumで実行 |
| 特定のファイル名 | 指定ファイルのテストをChromiumで実行 |

プロジェクトで定義されたE2Eテスト実行コマンドを使用すること。

### Phase 2: 結果分析

1. **テスト結果の確認**
   - パスしたテスト数
   - 失敗したテスト数
   - スキップされたテスト数

2. **失敗テストの分析**
   失敗したテストがある場合：
   - エラーメッセージを確認
   - 失敗の原因を特定
   - 修正方法を提案

### Phase 3: レポート

テスト結果を以下の形式で報告：

```
## E2Eテスト結果

### サマリー
- 総テスト数: X
- パス: X
- 失敗: X
- スキップ: X

### 失敗テスト（ある場合）
1. テスト名
   - エラー内容
   - 推奨対応

### 次のアクション
- （必要に応じて）
```

### Phase 4: 全ブラウザ実行の確認

Chromiumでのテストが完了したら、**必ずユーザーに確認する**：

> 「Chromiumでのテストが完了しました。Firefox、Webkitを含む全ブラウザでもテストを実行しますか？」

- **ユーザーが希望する場合**: 全ブラウザでE2Eテストを実行
- **不要な場合**: 終了

---

## トラブルシューティング

### よくあるエラーと対処法

| エラー | 対処法 |
|--------|--------|
| DB接続エラー | ローカルDBが起動しているか確認 |
| タイムアウト | 待機時間を延長、ネットワーク状態を確認 |
| 要素が見つからない | `data-testid` がコンポーネントに設定されているか確認 |
| 認証エラー | テストユーザーの設定を確認 |

### デバッグが必要な場合

UIモードやデバッグモードでE2Eテストを再実行してステップごとに確認する。
